USER 저장
* 회원가입은 firebase 이용하고 있음. 
* user 확인만 firebase로 하고, user 정보는 json-server 이용할 예정
  - firebase 사용이유 :  현재 로그인 중인지 아닌지 확인 가능
  - 변하지 않는(혹은 거의 변할일 없는) uid, email, password, 사용자 이름 
  - 서버를 두개나 쓰는거라...찝찝하긴한데 일단..
* 회원가입승인시 -> userCredential 받아올 때 json-server user도 업데이트!!
  - userCredential에서 받아올 것 : uid, email, (이름)
  - 더 넣어줄거 : image, topCetagory id{midCategoryname, midCategoryid, midCategoryDesc,}
